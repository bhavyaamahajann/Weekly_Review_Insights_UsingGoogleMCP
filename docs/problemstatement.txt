# Weekly Product Review Pulse & Fee Explainer for Groww

## Product Selected

Groww

## Overview

This project builds an AI-powered workflow that converts public customer reviews into a structured weekly product pulse and generates a standardized fee explanation for customer support and internal stakeholders.

The solution uses Large Language Models (LLMs) for theme discovery, quote extraction, summarization, and fee explanation generation. All document creation and email operations are performed through MCP (Model Context Protocol) servers, ensuring approval-gated actions and avoiding direct Google API integration.

---

# Problem Statement

Product, Support, and Leadership teams frequently rely on app reviews to understand customer sentiment and identify recurring issues. However, manually collecting reviews, identifying trends, extracting insights, and distributing reports is time-consuming.

The objective of this project is to automate the generation and delivery of a weekly customer feedback pulse for Groww using public Google Play Store reviews from the last 8–12 weeks.

The system transforms raw review data into:

* Theme clusters
* Representative customer quotes
* Actionable product recommendations
* A concise weekly stakeholder update
* A standardized fee explanation

The generated outputs are stored and distributed using MCP-powered Google Workspace integrations.

# Part A — Weekly Product Pulse

## Input

Public Google Play Store reviews collected during the previous 8–12 weeks.

## Processing Workflow

### Step 1: Review Ingestion

The system retrieves reviews and stores:

* Review text
* Rating
* Review date

### Step 2: Data Cleaning

* Remove duplicates
* Remove empty reviews
* PII scrubbing
* Normalize text

### Step 3: Theme Discovery

Reviews are converted into embeddings and grouped using clustering techniques.

Maximum themes generated:

* 5

The system identifies:

* Top 3 themes by volume and importance

Expected themes include:

1. App Performance
2. Withdrawal Experience
3. Customer Support

### Step 4: Quote Extraction

The system extracts:

* 3 real user quotes

Validation rules:

* Quotes must exist verbatim in review text
* No generated quotes
* No PII

### Step 5: Weekly Summary Generation

The LLM generates:

* ≤250-word weekly pulse
* Top findings
* Customer sentiment summary

### Step 6: Action Recommendations

Generate:

* 3 actionable ideas

Examples:

* Improve withdrawal transparency
* Reduce app crashes during peak usage
* Improve support response tracking

# Part B — Fee Explainer

## Fee Scenario Selected

Mutual Fund Exit Load

## Objective

Generate a structured, factual explanation suitable for customer support and internal knowledge-sharing.

## Requirements

* Maximum 6 bullets
* Neutral tone
* Facts only
* No recommendations
* No comparisons
* Include source references
* Include verification date

## Example Output Structure

### Mutual Fund Exit Load

* Exit load is a fee charged when units are redeemed before a scheme-defined holding period.
* Exit load conditions vary by mutual fund scheme.
* The fee is generally expressed as a percentage of redemption value.
* Exit load, when applicable, is deducted during redemption.
* Details are specified in the Scheme Information Document (SID).
* Investors should refer to the latest scheme documents for current exit load rules.

Last Checked: June 2026

Official Sources:

1. Groww Help Center
2. Groww Exit Load Information Page

# Idempotency

The system prevents duplicate runs by storing:

* Product Name
* ISO Week
* Document Section ID
* Email Draft ID

Re-running the same week updates the existing section rather than creating duplicates.

---

# Auditability

Each run stores:

* Timestamp
* Product
* ISO Week
* Generated report
* Doc section identifier
* Email draft identifier

This enables answering:

"What was sent, when, and for which week?"
